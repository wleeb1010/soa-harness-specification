## Validator choreography

```
SOA_HANDLER_KEYPAIR_OVERLAP_DIR=test-vectors/handler-keypair-overlap/
RUNNER_TEST_CLOCK=2026-04-22T12:00:00Z       # inside overlap
```

Validator submits two PDAs, one signed by each kid; both MUST return `handler_accepted: true`.

Edge-case scenarios (optional follow-up probes):
- Clock `2026-04-21T00:00:00Z` (before key-2 issued) → key-2 rejected, key-1 accepted.
- Clock `2026-04-24T00:00:00Z` (past key-1 retirement) → key-1 rejected, key-2 accepted.

## Production guard

`SOA_HANDLER_KEYPAIR_OVERLAP_DIR` is a test-only env hook per §10.6.2. Runner MUST refuse startup when set AND listener binds to a non-loopback interface.

## Deterministic regeneration

Seeds (32-byte ASCII, used as Ed25519 private-key material via PKCS#8 wrapping):

```
key-1: SOA-OVERLAP-HANDLER-KEY-1-v1.0!!
key-2: SOA-OVERLAP-HANDLER-KEY-2-v1.0!!
```

PureEdDSA signatures are canonical, so regenerated bytes are identical. Drift in `private.pem` or `spki_sha256.txt` indicates seed derivation changed.
